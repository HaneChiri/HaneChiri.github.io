文件相关

哪个是源程序？

Q：群文件里面这些文件哪个是源程序？

目录：

  • .vs
  • Debug

  • 项目名字

    • Debug
    • xxx.h
    • xxx.cpp
    • xxx.vcxproj
    • ……

  • 项目名字.sln

A：

  • .sln，这个是solution的缩写，是vs的解决方案文件，可以用vs打开。
  • .cpp或者.c，学过的都知道这是源文件吧？

怎么没有.sln文件

Q：为什么群文件里面的xx项目会没有SLN文件啊

A：大概率是因为不是用vs写的

界面相关

vs项目属性在哪里

在【解决方案资源管理器】中的你的项目名字上右击，在弹出菜单中找到【属性】，打开之后就是属性窗口

编译错误

以下所有错误在vs里面都可以直接点击错误编码跳转到官方文档。

【C1010】pch.h或stdafx.h

Q：群主能不能发一下pch.h/stdafx.h文件啊？没有这两个文件编译不了。

C1010 在查找预编译头时遇到意外的文件结尾。是否忘记了向源中添加“#include “stdafx.h”

A：请先百度这两个头文件，你会发现这俩是预编译头。用于加速编译。

解决方法：

  1. 打开项目属性窗口
  2. 【C/C++】->【预编译头】->【预编译头：不使用预编译头】
  3. 删去源代码中的stdafx.h或pch.h（如果需要）

参考链接：

  • 关于stdafx.h的一些解释-CSDN

【C4996】This function or variable may be unsafe

C4996 ‘strcpy’: This function or variable may be unsafe. Consider using strcpy_s instead. To disable deprecation, use _CRT_SECURE_NO_WARNINGS. See online help for details.

这个问题的主要特征是警告说xxx函数不安全，建议改成xxx_s函数，这个是因为项目开启了SDL检查（安全生命周期检查），这个检查会把一些原本是警告的问题当成错误，不让你编译成功。做一些大型项目的时候会用到，做些课设什么的没必要开着。

解决方案：

  1. 打开项目属性窗口
  2. 【C/C++】->【常规】->【SDL检查：否】

改完之后，以上错误会变成警告，可以编译通过，不必管。
